How the site experience is structured.

The current static build already models the core product flow: page routing, login gating, add-to-cart actions, total pricing, payment methods, and saved locations.

AuthUsers can skip at first, but cart and checkout still require login or sign up.
PaymentCard and UPI are supported. UPI triggers a request message, and COD is disabled.

System 01

Cart logic

  • Product prices stored on each card
  • Cart quantity and totals handled in shared client-side state
  • Checkout route connected to payment page

System 02

Location logic

  • Multiple saved locations supported
  • Labels must be unique
  • Home and work addresses cannot be the same

System 03

Theme control

  • Theme switch persists user choice
  • Entire site updates via shared variables
  • Cloud, Obsidian, and Sunset modes available

System 04

Commerce policy

  • Free shipping across India
  • Remote areas excluded
  • No cash on delivery
